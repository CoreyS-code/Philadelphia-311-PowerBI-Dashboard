# Philadelphia 311 Power BI Dashboard

## Project Overview

This project analyzes Philadelphia 311 service request data using Power BI to identify operational trends, agency workloads, and service resolution performance. The dashboard transforms raw service request data into actionable operational insights through interactive visualizations, data modeling, and DAX calculations.

## Dataset

**Source:** Philadelphia Open Data – 311 Service Requests

The dataset contains public records of non-emergency service requests submitted through the Philadelphia 311 system, including request dates, responsible agencies, service categories, and completion dates.

## Objectives

- Analyze service request volume across city agencies.
- Measure average resolution time for completed requests.
- Compare agency workload with resolution performance.
- Identify operational outliers and potential bottlenecks.

## Dashboard Pages

### Executive Overview

Provides a high-level summary of Philadelphia 311 operations, including:

- Total service requests
- Average resolution time
- Key performance indicators (KPIs)
- Overall operational trends

### Operational Insights

Explores agency performance through:

- Top agencies by request volume
- Top agencies by average resolution time
- Scatter plot comparing agency workload versus average resolution time

## Key Findings

- The Philadelphia 311 Contact Center processed the highest volume of service requests.
- The Police Department exhibited the longest average resolution time despite handling substantially fewer requests than the highest-volume agencies.
- Comparing workload against resolution time suggests that factors beyond request volume may contribute to longer case resolution times for certain agencies.
